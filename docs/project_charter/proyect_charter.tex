\section{Fase de Inicio: Project Charter}

El presente documento constituye el Acta de Constitución del Proyecto (Project Charter), sirviendo como marco formal para autorizar el desarrollo del Sistema de Gestión de Almacenes (WMS) para CO LOGISTIC S.A.C. y definir sus objetivos, alcance, recursos, roles y riesgos principales.

\subsection{Datos Identificatorios del Proyecto}
\begin{itemize}[label=\textbullet]
  \item \textbf{Razón Social del Cliente:} CO LOGISTIC S.A.C. (Empresa de servicios logísticos, almacenamiento y distribución).
  \item \textbf{Contacto del Cliente:} Gerencia de Operaciones y Logística.
  \item \textbf{Razón Social del Consultor:} Equipo de Proyecto --- Escuela Profesional de Ingeniería de Sistemas, Universidad Peruana Unión (UPeU).
  \item \textbf{Contacto del Consultor:} Comité de Desarrollo y Gobierno del Proyecto.
\end{itemize}

\subsection{Bases Legales y Normativas}
Para garantizar los estándares técnicos, de seguridad y la confidencialidad en el tratamiento de la información, se consideran los siguientes marcos normativos y regulatorios:
\begin{itemize}[label=\textbullet]
  \item \textbf{Estándar IEEE 29148:} Adoptado como directriz formal para la Ingeniería de Requisitos y la correcta especificación de requerimientos de software (SRS).
  \item \textbf{Norma ISO/IEC 27001:} Aplicada para garantizar políticas de seguridad de la información, control de accesos al sistema, privilegios de usuarios y cifrado de credenciales de los clientes.
  \item \textbf{Código de Ética ACM:} Garantizando la estricta confidencialidad en el tratamiento de los datos logísticos de los clientes de CO LOGISTIC S.A.C. (tales como Bimbo u otros socios corporativos).
\end{itemize}

---

\section{Objetivos del Project Charter}

\subsection{Objetivo General}
Implementar un Sistema de Gestión de Almacenes (WMS) bajo una arquitectura de microservicios e integración de Machine Learning para estandarizar los procesos de recepción, almacenamiento y despacho de mercadería, y proyectar de forma inteligente la capacidad operativa y ocupación estacional de la empresa CO LOGISTIC S.A.C. en un periodo estimado de 6 meses.

\subsection{Objetivos Específicos}
\begin{itemize}[label=\textbullet]
  \item Desarrollar los módulos transaccionales del almacén (Ingreso, Ubicación y Salida) utilizando el framework \textbf{Spring Boot} para una alta disponibilidad.
  \item Implementar un modelo de Machine Learning (mediante algoritmos de regresión o análisis de series de tiempo) para pronosticar la ocupación del almacén con base en registros históricos.
  \item Reducir los errores de descuadre de inventario y mejorar la ubicabilidad de productos mediante validaciones automatizadas y control en tiempo real durante los despachos.
\end{itemize}

---

\section{Objetivos del Proyecto Tecnológico}

\subsection{Objetivo General del Proyecto}
Diseñar e implementar un Sistema de Gestión de Almacenes (WMS) desde cero, bajo una arquitectura moderna de microservicios, integrado con un modelo predictivo de Machine Learning (forecasting) para optimizar la logística de inventario, ubicabilidad y despachos de CO LOGISTIC S.A.C.

\subsection{Objetivos Específicos del Proyecto}
\begin{itemize}[label=\textbullet]
  \item Desarrollar el núcleo transaccional de logística de inventario (ingresos, ubicaciones, despachos) utilizando Spring Boot en Java.
  \item Entrenar e integrar un modelo de regresión o clasificación en Python alimentado por los datos históricos transaccionales del WMS para pronosticar la capacidad estacional del almacén.
  \item Justificar y aplicar el marco de trabajo Scrum para la planificación técnica, la priorización de historias de usuario y la gestión del Product Backlog en Sprints.
  \item Evaluar el rendimiento, la precisión y seguridad de las consultas generadas y la base de datos de pruebas mediante auditorías SQL.
\end{itemize}

\section{Justificación del Proyecto}

La implementación de este proyecto es fundamental para modernizar el núcleo operativo de CO LOGISTIC S.A.C. Al desarrollar un Sistema de Gestión de Almacenes (WMS) a la medida, la empresa automatizará y estandarizará sus procesos transaccionales (ingresos, ubicaciones y despachos), eliminando errores manuales y generando una base de datos histórica confiable. Sobre esta data, la integración del modelo predictivo de Machine Learning (forecasting) permitirá proyectar la capacidad y ocupación estacional del almacén, brindando a la gerencia una herramienta estratégica para anticipar la disponibilidad de espacios y maximizar la rentabilidad del negocio.

---

\section{Descripción del Beneficiario del Proyecto}

Los beneficiarios directos del proyecto incluyen a la gerencia y el personal operativo de CO LOGISTIC S.A.C., quienes contarán con una herramienta moderna y automatizada que reducirá la carga operativa, eliminará procesos manuales y agilizará la toma de decisiones. Asimismo, los clientes arrendadores de la empresa (como Bimbo) se beneficiarán indirectamente al recibir un servicio logístico de mayor precisión, rapidez, y trazabilidad de sus productos.

---

\section{Origen del Proyecto}

El proyecto de WMS surge ante la necesidad operativa de digitalizar el control logístico, el cual actualmente se realiza de manera empírica o mediante herramientas limitadas como archivos Excel individuales. El problema central radica en que, sin un sistema transaccional robusto, la empresa carece de métricas exactas en tiempo real y de datos históricos estructurados. Esta falta de digitalización impide realizar pronósticos sobre la saturación del almacén en temporadas altas, lo que genera cuellos de botella en la operación logística y limita la capacidad de los gerentes para tomar decisiones estratégicas sobre el arrendamiento futuro de los espacios disponibles.

---

\section{Objetivos del Producto que Generará el Proyecto}

\subsection{Objetivo General del Producto}
Digitalizar y centralizar la gestión de inventario de CO LOGISTIC S.A.C. mediante un sistema transaccional robusto, y dotar a la gerencia de una herramienta predictiva (basada en Machine Learning) que permita anticipar la capacidad y ocupación estacional del almacén para una mejor toma de decisiones.

---

\section{Alcance del Proyecto}

El proyecto abarcará el ciclo de vida completo del desarrollo de software (diseño, implementación y pruebas). Incluirá la construcción del backend bajo una arquitectura de microservicios (utilizando Spring Boot), el diseño e implementación de la base de datos operacional con permisos completos (CRUD/DML), el desarrollo del frontend web (dashboard y módulos operativos), y la creación de un microservicio en Python dedicado a ejecutar el modelo predictivo de Machine Learning.

No incluirá el desarrollo de módulos ajenos a la logística de almacén (como facturación, contabilidad o recursos humanos), ni abarcará la integración con hardware externo especializado.
